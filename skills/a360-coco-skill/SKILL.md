# Account 360 (A360) Skill

Answer questions about Snowflake customer accounts using accessible internal data sources.

## Key Principle

All views join on `SALESFORCE_ACCOUNT_ID`. Always filter by this column when querying for a specific account. Users may provide account name, which requires a lookup first.

## Account Identification

When user provides an account name (not ID), first find the SALESFORCE_ACCOUNT_ID:

```sql
SELECT DISTINCT 
    ID as SALESFORCE_ACCOUNT_ID, 
    NAME as ACCOUNT_NAME, 
    INDUSTRY, 
    TYPE as ACCOUNT_TYPE,
    BILLING_COUNTRY
FROM FIVETRAN.SALESFORCE.ACCOUNT
WHERE LOWER(NAME) LIKE LOWER('%<search_term>%')
  AND IS_DELETED = FALSE
LIMIT 10;
```

If multiple matches, ask user to clarify which account.

## Query Patterns

### 1. Account Profile

```sql
SELECT 
    a.ID as SALESFORCE_ACCOUNT_ID,
    a.NAME as ACCOUNT_NAME,
    a.INDUSTRY,
    a.TYPE as ACCOUNT_TYPE,
    a.ANNUAL_REVENUE,
    a.NUMBER_OF_EMPLOYEES,
    a.BILLING_COUNTRY,
    a.WEBSITE,
    o.SALESFORCE_ACCOUNT_OWNER_NAME as ACCOUNT_OWNER,
    o.SALESFORCE_ACCOUNT_REGIONAL_DIRECTOR as RD,
    o.SALESFORCE_ACCOUNT_REGIONAL_VICE_PRESIDENT as RVP,
    o.SALESFORCE_ACCOUNT_GEO_VICE_PRESIDENT as GVP,
    o.SALES_GEO,
    o.SALES_TERRITORY,
    o.SALES_SEGMENT
FROM FIVETRAN.SALESFORCE.ACCOUNT a
LEFT JOIN SNOWHOUSE.SALES.SALESFORCE_ACCOUNT_OWNERSHIP o 
    ON a.ID = o.SALESFORCE_ACCOUNT_ID
WHERE a.ID = '<id>';
```

### 2. Revenue Trend (Daily)

```sql
SELECT 
    GENERAL_DATE,
    SUM(COMPUTE_CREDITS) as COMPUTE_CREDITS,
    SUM(CLOUD_SERVICES_CREDITS) as CLOUD_SERVICES_CREDITS,
    SUM(SNOWPIPE_CREDITS) as SNOWPIPE_CREDITS,
    SUM(AI_SERVICES_CREDITS) as AI_SERVICES_CREDITS,
    SUM(TOTAL_CREDITS) as TOTAL_CREDITS,
    SUM(GAAP_REVENUE) as GAAP_REVENUE
FROM FINANCE.CUSTOMER.CONTRACT_REVENUE
WHERE SALESFORCE_ACCOUNT_ID = '<id>'
  AND GENERAL_DATE >= DATEADD(day, -90, CURRENT_DATE())
  AND IS_FUTURE = FALSE
GROUP BY GENERAL_DATE
ORDER BY GENERAL_DATE DESC;
```

### 3. Revenue Trend (Monthly)

```sql
SELECT 
    DATE_TRUNC('month', GENERAL_DATE) as MONTH,
    SUM(COMPUTE_CREDITS) as COMPUTE_CREDITS,
    SUM(AI_SERVICES_CREDITS) as AI_SERVICES_CREDITS,
    SUM(TOTAL_CREDITS) as TOTAL_CREDITS,
    SUM(GAAP_REVENUE) as GAAP_REVENUE
FROM FINANCE.CUSTOMER.CONTRACT_REVENUE
WHERE SALESFORCE_ACCOUNT_ID = '<id>'
  AND GENERAL_DATE >= DATEADD(month, -6, CURRENT_DATE())
  AND IS_FUTURE = FALSE
GROUP BY DATE_TRUNC('month', GENERAL_DATE)
ORDER BY MONTH DESC;
```

### 4. Workload Distribution

```sql

    MONTH,
    SUM(DATA_LAKE_CREDITS) as DATA_LAKE,
```

### 10. New Tools/Connectors (last N days)

```sql
WITH sf_accounts AS (
    SELECT DISTINCT SNOWFLAKE_ACCOUNT_ID as ACCOUNT_ID
    FROM FINANCE.CUSTOMER.CONTRACT_REVENUE
    WHERE SALESFORCE_ACCOUNT_ID = '<id>' AND GENERAL_DATE >= DATEADD(day, -90, CURRENT_DATE()) AND IS_FUTURE = FALSE
),
tool_first_seen AS (
    SELECT TOOL, CLIENT, MIN(END_DATE) as first_seen
    FROM SNOWSCIENCE.JOB_ANALYTICS.JOB_CREDITS_DAY_TOOL_FACT j
    JOIN sf_accounts s ON j.ACCOUNT_ID = s.ACCOUNT_ID
    GROUP BY TOOL, CLIENT
)
SELECT TOOL, CLIENT, first_seen FROM tool_first_seen
WHERE first_seen >= DATEADD(day, -14, CURRENT_DATE())
ORDER BY first_seen DESC;
```

### 17. Month-over-Month Growth

```sql
WITH monthly AS (
    SELECT DATE_TRUNC('month', GENERAL_DATE) as MONTH, SUM(TOTAL_CREDITS) as TOTAL_CREDITS, SUM(GAAP_REVENUE) as GAAP_REVENUE
    FROM FINANCE.CUSTOMER.CONTRACT_REVENUE
    WHERE SALESFORCE_ACCOUNT_ID = '<id>' AND IS_FUTURE = FALSE
    GROUP BY DATE_TRUNC('month', GENERAL_DATE)
)
SELECT MONTH, TOTAL_CREDITS, GAAP_REVENUE, LAG(TOTAL_CREDITS) OVER (ORDER BY MONTH) as PREV_MONTH_CREDITS,
    ROUND((TOTAL_CREDITS - LAG(TOTAL_CREDITS) OVER (ORDER BY MONTH)) / NULLIF(LAG(TOTAL_CREDITS) OVER (ORDER BY MONTH), 0) * 100, 1) as MOM_GROWTH_PCT
FROM monthly ORDER BY MONTH DESC LIMIT 13;
```

## Workflow

1. **Identify Account**: If user provides name, look up SALESFORCE_ACCOUNT_ID first using `FIVETRAN.SALESFORCE.ACCOUNT`
2. **Determine Intent**: Match question to appropriate data source and query pattern
3. **Build Query**: Use patterns above, adjust date ranges as needed
4. **Execute & Explain**: Run query, summarize findings in plain language

## Stopping Points

- **Multiple account matches**: If account name lookup returns >1 result, ask user to clarify which account before proceeding
- **Ambiguous time range**: If user doesn't specify date range for trends/usage, confirm default (e.g., L30D, L90D) or ask preference
- **No data found**: If query returns empty results, inform user and offer to check alternative sources or time ranges

## Output

Responses should include:
- **Summary**: Plain language interpretation of the data
- **Key metrics**: Formatted tables or bullet points with relevant numbers
- **Trends**: Highlight significant changes (growth, decline, anomalies)
- **Context**: Compare to prior periods when relevant (MoM, YoY)

## Notes

- All sources join on `SALESFORCE_ACCOUNT_ID` except tool tables which use `ACCOUNT_ID` (Snowflake account ID)
- Map Snowflake to Salesforce account via `FINANCE.CUSTOMER.CONTRACT_REVENUE`
- Support case JSON structure: `CASE_STRUCTURED:metadata:case:<field>` (e.g., `sfdc_account_id`, `status`, `severity`, `created_at`)
- Default to L30D for product adoption queries, L6M for revenue trends unless user specifies otherwise
- Always filter `IS_FUTURE = FALSE` on CONTRACT_REVENUE to exclude forecast data
