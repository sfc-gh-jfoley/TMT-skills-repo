### PHASE 2.5: Account Manager Perspective Enrichment (CRITICAL)

**PURPOSE**: These additional queries capture the signals a seasoned Account Manager looks for to understand account health, engagement patterns, and expansion signals.

#### Step 2.13: Win/Loss History Analysis
```sql
-- Understanding win/loss patterns reveals competitive dynamics and sales effectiveness
SELECT 
    FISCAL_YEAR,
    OPPORTUNITY_TYPE,
    STAGE_NAME,
    COUNT(*) as OPP_COUNT,
    SUM(TOTAL_ACV) as TOTAL_ACV,
    SUM(CASE WHEN IS_WON = TRUE THEN 1 ELSE 0 END) as WINS,
    SUM(CASE WHEN IS_LOST = TRUE THEN 1 ELSE 0 END) as LOSSES,
    SUM(CASE WHEN IS_WON = TRUE THEN TOTAL_ACV ELSE 0 END) as WON_ACV,
```

#### Step 2.15: Consumption Velocity & Trends
```sql
-- Month-over-month growth rate reveals expansion momentum or contraction risk
SELECT 
    DATE_TRUNC('MONTH', GENERAL_DATE::DATE) AS MONTH,
    SUM(REVENUE) AS MONTHLY_REVENUE,
    LAG(SUM(REVENUE)) OVER (ORDER BY DATE_TRUNC('MONTH', GENERAL_DATE::DATE)) as PRIOR_MONTH,
    ROUND(((SUM(REVENUE) - LAG(SUM(REVENUE)) OVER (ORDER BY DATE_TRUNC('MONTH', GENERAL_DATE::DATE))) / 
           NULLIF(LAG(SUM(REVENUE)) OVER (ORDER BY DATE_TRUNC('MONTH', GENERAL_DATE::DATE)), 0)) * 100, 2) as MOM_GROWTH_PCT
FROM SALES.RAVEN.A360_DAILY_ACCOUNT_PRODUCT_CATEGORY_REVENUE_VIEW
WHERE SALESFORCE_ACCOUNT_ID = '{SALESFORCE_ACCOUNT_ID}'
  AND GENERAL_DATE >= DATEADD(month, -18, CURRENT_DATE())
GROUP BY 1
ORDER BY 1 DESC;
```

#### Step 2.16: Product Category Growth Analysis
```sql
-- Which products are growing vs declining? This reveals expansion vs contraction areas
SELECT 
    PRODUCT_CATEGORY,
    SUM(CASE WHEN GENERAL_DATE >= DATEADD(month, -3, CURRENT_DATE()) THEN REVENUE ELSE 0 END) as LAST_3M,
    SUM(CASE WHEN GENERAL_DATE >= DATEADD(month, -6, CURRENT_DATE()) AND GENERAL_DATE < DATEADD(month, -3, CURRENT_DATE()) THEN REVENUE ELSE 0 END) as PRIOR_3M,
    ROUND(((SUM(CASE WHEN GENERAL_DATE >= DATEADD(month, -3, CURRENT_DATE()) THEN REVENUE ELSE 0 END) - 
            SUM(CASE WHEN GENERAL_DATE >= DATEADD(month, -6, CURRENT_DATE()) AND GENERAL_DATE < DATEADD(month, -3, CURRENT_DATE()) THEN REVENUE ELSE 0 END)) /
           NULLIF(SUM(CASE WHEN GENERAL_DATE >= DATEADD(month, -6, CURRENT_DATE()) AND GENERAL_DATE < DATEADD(month, -3, CURRENT_DATE()) THEN REVENUE ELSE 0 END), 0)) * 100, 2) as QOQ_GROWTH_PCT
FROM SALES.RAVEN.A360_DAILY_ACCOUNT_PRODUCT_CATEGORY_REVENUE_VIEW
WHERE SALESFORCE_ACCOUNT_ID = '{SALESFORCE_ACCOUNT_ID}'
GROUP BY PRODUCT_CATEGORY
HAVING LAST_3M > 0 OR PRIOR_3M > 0
ORDER BY LAST_3M DESC;
```
